# ITERO: Менеджер IT проєктів з дошкою Kanban

ITERO — це повнофункціональний десктопний менеджер для управління проєктами за методологією Kanban з підтримкою drag-and-drop переміщення завдань, редагування деталей та управління файловими вкладеннями.

## Вимоги до системи

Перед встановленням переконайтесь, що у вас встановлені:

- **Node.js** >= 20.0.0 ([Завантажити](https://nodejs.org/))
- **npm** >= 9.0.0 (встановлюється разом з Node.js)
- **Git** ([Завантажити](https://git-scm.com/))

**Системні вимоги:**
- RAM: >= 2GB
- Дискове місце: >= 500MB
- ОС: Windows, macOS, Linux

## Встановлення

### Крок 1: Клонування репозиторію

```bash
git clone https://github.com/MikeSydo/Itero.git
cd Itero
```

### Крок 2: Встановлення залежностей

```bash
npm install
```

### Крок 3: Налаштування оточення

Створіть файл `.env` у корені проєкту:

```bash
cp .env.example .env
```

Якщо файл `.env.example` не існує, створіть `.env` вручну з наступним вмістом:

```env
PORT=3000
NODE_ENV=development
DATABASE_URL="file:./dev.db"
```

### Крок 4: Ініціалізація бази даних

```bash
npx prisma migrate dev
```

Це створить базу даних SQLite та виконає всі необхідні міграції.

### Крок 5: Завантаження тестових даних (опціонально)

```bash
npx prisma db seed
```

## Запуск

### Режим розробки (все разом)

```bash
npm run dev
```

Це запустить одночасно:
- Express.js сервер на **http://localhost:3000**
- React dev сервер на **http://localhost:8000**
- Electron додаток (десктопне вікно)

### Запуск окремих компонентів

```bash
npm run dev:api       # Тільки Express сервер
npm run dev:client    # Тільки React фронтенд
npm run dev:electron  # Тільки Electron додаток
```

### Production режим

#### Збирання проєкту

```bash
npm run build
```

#### Запуск production сервера

```bash
npm run start:api
```

#### Запуск десктопної версії

```bash
npm run start:electron
```

### Упаковка для розповсюдження

```bash
npm run make
```

Це створить установники у директорії `out/make`:
- **Windows**: `ITERO Setup.exe`
- **macOS**: `ITERO.dmg`
- **Linux**: `itero-1.0.0.AppImage`

## Користування системою

### Створення дошки

1. Клікніть на кнопку `+ New Board`
2. Введіть назву дошки
3. Клікніть `Create`

### Створення списку

1. Перейдіть до дошки
2. Клікніть на `+ New List`
3. Введіть назву списку
4. Клікніть `Create`

### Створення завдання

1. Перейдіть до списку
2. Клікніть на `+ Create`
3. Введіть назву завдання
4. Натисніть Enter

### Перетягування завдань

1. Утримуйте ліву кнопку миші на завданні
2. Перетягніть завдання в інший список чи позицію
3. Відпустіть кнопку миші

### Редагування завдання

1. Клікніть на завдання для відкриття деталей
2. Відредагуйте назву, опис, дати, колір
3. Зміни зберігаються автоматично

### Додавання файлів

1. Відкрийте завдання
2. Клікніть на `Додати файл`
3. Виберіть файл з комп'ютера
4. Файл завантажиться автоматично (макс. 10MB)

### Управління дошками

- **Додати до улюблених**: клік на іконку favorite на картці дошки
- **Перейменування**: двійний клік на назву дошки
- **Видалення**: клік на іконку видалення (видалить всі списки та завдання)

## Вирішення проблем

### Білий екран при запуску

Переконайтесь, що:
- React dev сервер запущений на порту **8000**
- Express сервер запущений на порту **3000**
- База даних створена: `npx prisma migrate dev`

### Помилка підключення до БД

1. Перевірте файл `.env`
2. Переконайтесь, що директорія для БД існує
3. Запустіть: `npx prisma migrate dev`

### Помилка портів (port already in use)

Якщо порти 3000 або 8000 вже зайняті, змініть їх у файлі `.env`:

```env
PORT=3001
```

Також оновіть URL сервера у коді фронтенду, якщо змінили порт.

### Файли не завантажуються

- Переконайтесь, що директорія `uploads` існує та має права для запису
- Максимальний розмір файлу — 10MB
- Перевірте логи сервера на помилки

## Розробка

### Структура проєкту

```
itero/
├── src/                      # Код фронтенду
│   ├── components/          # React компоненти
│   ├── pages/               # Сторінки додатку
│   ├── hooks/               # React гачки
│   └── styles/              # CSS файли
├── api/                     # Код бекенду
│   ├── routes/              # API endpoints
│   ├── middleware/          # Express middleware
│   └── server.ts            # Основний файл сервера
├── prisma/                  # Схема та міграції БД
│   ├── schema.prisma        # Опис моделей
│   └── migrations/          # Історія змін БД
├── electron/                # Конфігурація Electron
├── uploads/                 # Директорія для завантажених файлів
├── package.json             # Залежності та скрипти
└── .env                     # Конфігураційні змінні
```

### Prisma Studio для управління БД

Для візуального управління базою даних:

```bash
npx prisma studio
```

Це відкриє веб-інтерфейс на **http://localhost:5555**

### Перегляд логів сервера

Логи Express сервера виводяться в консоль, де він запущений. Кожен запит логується з датою, методом та шляхом.

### Тестування API

Для тестування API можна використовувати:
- [Postman](https://www.postman.com/)
- [Insomnia](https://insomnia.rest/)
- [Thunder Client](https://www.thunderclient.com/)

### Основні API endpoints

**Дошки:**
- `GET /boards` — отримати всі дошки
- `POST /boards` — створити дошку
- `PATCH /boards/:id` — оновити дошку
- `DELETE /boards/:id` — видалити дошку

**Списки:**
- `GET /lists` — отримати всі списки
- `POST /lists` — створити список
- `PATCH /lists/:id` — оновити список
- `DELETE /lists/:id` — видалити список

**Завдання:**
- `GET /tasks` — отримати всі завдання
- `POST /tasks` — створити завдання
- `PATCH /tasks/:id` — оновити завдання
- `DELETE /tasks/:id` — видалити завдання

## Резервна копія даних

Вся база даних зберігається в одному файлі `dev.db`. Для резервної копії просто скопіюйте цей файл:

```bash
cp dev.db dev.db.backup
```
